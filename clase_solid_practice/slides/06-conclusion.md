## Aplicar SOLID para Refactorizar

| Antes | Después |
|:-----:|:-------:|
| Una clase con muchas responsabilidades | Múltiples clases enfocadas |
| Alto acoplamiento | Bajo acoplamiento |
| Difícil de testear | Fácil de mockear |
<!-- .element: style="font-size: 0.85em;" -->

--

## Patrones de Diseño Utilizados

| Patrón | Uso |
|:------:|:----|
| **Strategy** | Algoritmos intercambiables |
| **Factory** | Creación de objetos |
| **Chain of Responsibility** | Procesamiento secuencial |
| **Observer** | Manejo de eventos |
| **Dependency Injection** | Bajo acoplamiento |
<!-- .element: style="font-size: 0.8em;" -->

--

## Beneficios de Testing

| Con SOLID | Sin SOLID |
|:---------:|:---------:|
| Mock de cada componente | Mocks complejos |
| Testing en aislamiento | Tests integrados |
| Aserciones claras | Aserciones confusas |
| Coverage alto | Difícil cobertura |
<!-- .element: style="font-size: 0.85em;" -->
